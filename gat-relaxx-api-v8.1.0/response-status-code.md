# Response status code

The following HTTP status codes are used within the Relaxx v8.1.0 API.

|          |                                                                                                                                                                                                                                                                                                                                                |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Code** | **Description**                                                                                                                                                                                                                                                                                                                                |
| 200      | <p><strong>OK</strong></p><p>The request was successful, resulting in everything working as expected.</p>                                                                                                                                                                                                                                      |
| 201      | <p><strong>Created</strong></p><p>The request was successful, resulting in the creation of new resources.</p>                                                                                                                                                                                                                                  |
| 202      | <p><strong>Accepted</strong></p><p>The request has been accepted for processing, but the processing has not been completed.</p>                                                                                                                                                                                                                |
| 204      | <p><strong>No content</strong></p><p>The request was successfully processed and is not returning any content.</p>                                                                                                                                                                                                                              |
| 400      | <p><strong>Bad request</strong></p><p>The server was unable to understand the request. The request is most likely malformed or a mandatory parameter is missing.</p><p></p><p><strong>Note:</strong> It is recommended that modifications to the current request be made and the request be repeated.</p>                                      |
| 401      | <p><strong>Unauthorized</strong></p><p>The request has not been processed because it lacks valid authentication credentials for the target resource. <br><br><strong>Note:</strong> If the request included valid authentication credentials, then the 401 response indicates that authorization has been refused for the target resource.</p> |
| 403      | <p><strong>Forbidden</strong></p><p>The request contains valid authentication credentials but will not be processed because of a lack of permission.</p>                                                                                                                                                                                       |
| 404      | <p><strong>Not found</strong></p><p>The server understood the request but refused to authorize it. </p><p></p><p><strong>Note:</strong> The resource may be available in the future. Subsequent requests by the client are permissible.</p>                                                                                                    |
| 409      | <p><strong>Conflict</strong></p><p>The request was valid, but the current system state does not permit this action.</p>                                                                                                                                                                                                                        |
| 422      | <p><strong>Unacceptable entity</strong></p><p>The server understands the content type of the request entity but was unable to process the instructions contained in itc.</p><p></p><p>For example, this error condition may occur if validation rules to process the entity fail.</p>                                                          |
| 429      | <p><strong>Too many requests</strong></p><p>The server received too many requests in a given amount of time.</p>                                                                                                                                                                                                                               |
| 500      | <p><strong>Internal server error</strong></p><p>The server encountered an unexpected error, which prevented it from fulfilling the request.</p>                                                                                                                                                                                                |
| 502      | <p><strong>Bad gateway</strong></p><p>The server, while acting as a gateway proxy, received an invalid response from the upstream server.</p>                                                                                                                                                                                                  |
| 503      | <p><strong>Service unavailable</strong></p><p>The server is currently unable to handle the request due to a temporary overloading or maintenance of the server.</p>                                                                                                                                                                            |
